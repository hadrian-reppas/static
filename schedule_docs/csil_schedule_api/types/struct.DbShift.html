Expand description

A CSIL shift in the database

Conversions Into this type check that the date is in the valid range, the start and end times are in the valid range, the start and end times multiples of MINUTE_INCREMENT and the start time is stricly before the end time. This ensure that every shift is valid before it is put into the database.

In principle, this type can diverge from the Shift representation that the frontend and API use. For now, we just convert the [ChicagoTime]s into i32s and the [CleaningLocation]s into machine-readable Strings.
